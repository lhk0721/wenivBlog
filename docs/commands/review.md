# Review Guide

이 문서는 코드 리뷰 요청/수행 시 공통으로 따르는 일반 가이드다.

## 목적
- 버그, 회귀, 보안/성능 리스크를 우선 발견한다.

## 리뷰 우선순위
1. 기능 오동작/회귀 가능성
2. 예외/경계값 처리 누락
3. 보안 위험(인증/인가/입력검증/비밀값 노출)
4. 성능 저하 가능성
5. 테스트 누락/부족

## 체크리스트
- 요구사항을 정확히 충족하는가
- 코드 변경 범위가 과도하지 않은가
- 실패 케이스에 대한 방어 코드가 있는가
- 테스트가 실질적으로 회귀를 막는가
- 문서/주석/네이밍이 의도를 설명하는가

## 리뷰 결과 작성 형식
- Findings: 심각도 높은 순으로 나열
- 각 항목: 파일 경로 + 문제 + 영향 + 제안
- Open Questions: 확정이 필요한 가정/질문
- Summary: 변경 요약(간단히)

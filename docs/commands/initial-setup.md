# Initial Setup Guide

이 문서는 특정 프로젝트에 종속되지 않는 공통 초기 세팅 가이드다.

## 적용 원칙
- 먼저 현재 저장소 구조와 사용 기술 스택을 확인한다.
- 스캐폴딩은 최소 구조부터 만든다.
- 도메인 전용 화면명/경로는 예시로만 참고하고 사용자 요구를 우선한다.

## 기본 체크리스트
1. 기존 디렉터리/엔트리 파일 확인 (`src/main.*`, `src/App.*`)
2. 공통 폴더 생성 (`assets`, `components`, `layouts`, `pages`, `routes`)
3. 페이지 단위 파일 스캐폴딩 생성
4. 라우팅 사용 시 경로 상수와 라우터 엔트리 구성
5. 전역 스타일 파일 연결
6. 빌드 또는 최소 실행 검증

## 권장 폴더 구조 (예시)
```text
src/
  assets/
    icons/
    images/
    styles/
  components/
    common/
  layouts/
  pages/
  routes/
```

## 라우팅 가이드 (예시)
- 라우팅 라이브러리 필요 시 설치 후 적용
- `App`는 라우터 진입점 역할만 유지
- 라우트 경로는 `routes/paths.*`로 분리
- 페이지 라우트 구성은 `routes/AppRouter.*`로 분리
- 미정의 경로 처리(`*`)는 명시적으로 정의

## 검증 규칙
- 변경 후 `build` 또는 동등한 검증 명령 실행
- 실패 시 원인(권한/의존성/코드)을 구분해 보고
- 생성/수정 파일 목록을 함께 보고

## 도메인 예시 기록 방법
- 블로그, 커머스, 대시보드 등 도메인 예시는 별도 섹션으로 관리
- 예시는 기본값이 아니며, 사용자 요청 시에만 적용

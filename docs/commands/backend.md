# Backend Guide

이 문서는 백엔드 작업 시 공통으로 따르는 일반 가이드다.

## 목적
- API/비즈니스 로직/데이터 계층 변경을 안전하게 수행한다.

## 기본 절차
1. 요구사항과 입출력 계약(API 스펙) 확인
2. 영향 범위(라우트, 서비스, DB, 미들웨어) 파악
3. 변경 구현 및 예외 처리 추가
4. 테스트/정적 검증 실행

## 체크리스트
- 요청/응답 스키마가 일관적인가
- 에러 코드/메시지 규칙을 지켰는가
- 인증/인가가 필요한 엔드포인트가 보호되는가
- DB 변경 시 마이그레이션/롤백 경로가 있는가
- 로그에 민감정보가 노출되지 않는가

## 권장 검증
- 단위 테스트
- 통합 테스트
- 빌드/타입 체크(사용 중인 스택 기준)

## 보고 형식
- 변경 API/로직
- 데이터 영향도
- 검증 결과

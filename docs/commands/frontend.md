# Frontend Guide

이 문서는 프론트엔드 작업 시 공통으로 따르는 일반 가이드다.

## 목적
- 화면/상태/라우팅/스타일 변경 시 작업 범위를 명확히 한다.

## 기본 절차
1. 영향 범위 확인 (`pages`, `components`, `routes`, `styles`)
2. UI 구조와 상태 흐름을 먼저 정리
3. 필요한 파일만 최소 수정
4. 린트/빌드/동작 확인

## 체크리스트
- 컴포넌트 책임이 과도하지 않은가
- 재사용 가능한 부분을 공통 컴포넌트로 분리했는가
- 라우트/경로 상수와 실제 페이지 연결이 일치하는가
- 접근성(버튼/라벨/키보드 포커스) 기본 요건을 지켰는가
- 모바일/데스크톱에서 레이아웃 깨짐이 없는가

## 권장 검증
- `npm run lint`
- `npm run build`

## 보고 형식
- 변경 파일
- UI/동작 변경점
- 검증 결과

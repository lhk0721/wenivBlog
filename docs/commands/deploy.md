# Deploy Guide

이 문서는 배포 작업 시 공통으로 따르는 일반 가이드다.

## 목적
- 배포 과정의 누락/장애를 줄이고 재현 가능한 절차를 유지한다.

## 기본 절차
1. 배포 대상 버전/브랜치 확인
2. 환경 변수/시크릿/설정값 확인
3. 빌드 및 아티팩트 검증
4. 배포 실행
5. 헬스체크 및 롤백 준비 상태 확인

## 체크리스트
- 릴리스 노트(변경점/영향도) 준비 여부
- DB 마이그레이션 선후행 순서 확인
- 캐시/큐/스케줄러 영향 확인
- 모니터링/알람 대시보드 확인
- 장애 시 롤백 절차 확인

## 배포 후 확인
- 핵심 기능 스모크 테스트
- 에러율/지연시간/리소스 사용량 모니터링
- 사용자 영향 여부 확인

## 보고 형식
- 배포 버전/시간
- 수행 단계 요약
- 확인 결과 및 이슈

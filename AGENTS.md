# AGENTS.md

이 파일은 이 저장소에서 작업할 때 에이전트가 따라야 할 운영 규칙을 정의한다.

## 1) 응답/커뮤니케이션 규칙
- 기본 응답 언어: 한국어
- 보고 방식: 결과 먼저, 세부 설명은 뒤에
- 불확실한 내용은 추정이라고 명시
- 작업이 길어지면 중간 진행 상황을 짧게 공유

## 2) 코드 작업 기본 원칙
- 변경은 최소 범위로 수행
- 기존 코드 스타일/패턴 우선 준수
- 검색은 `rg` 우선 사용
- 불필요한 리팩터링 금지
- 요청되지 않은 라이브러리 추가 금지

## 3) 파일/구조 규칙
- 문서는 `docs/` 아래 관리
- 작업별 세부 지침은 `docs/commands/*.md`로 분리
- 환경별 설정은 예시 파일(`.env.example`) 기준으로 유지

## 4) 구현 절차 (기본 플로우)
1. 관련 파일과 영향 범위를 먼저 파악한다.
2. 변경 계획을 짧게 정리한다.
3. 필요한 파일만 수정한다.
4. 테스트/빌드/린트를 실행한다.
5. 변경 요약과 검증 결과를 보고한다.

## 5) 테스트/검증 규칙
- 코드 변경 시 관련 테스트를 우선 실행
- 테스트가 없으면 최소 동작 검증 절차를 문서화
- 실행 불가 시 이유와 대체 검증 방법을 명시

## 6) 금지 사항
- 비밀값/토큰/개인정보 커밋 금지
- 파괴적 명령(`reset --hard`, 대량 삭제 등) 임의 실행 금지
- 사용자 요청 없는 대규모 포맷팅 금지

## 7) 커밋 규칙 (선택)
- 커밋 메시지는 Conventional Commits 권장
- 예시: `feat: add post list filtering`
- 하나의 커밋에는 하나의 논리적 변경만 포함

## 8) 작업 완료 보고 형식
- 변경 파일: `path/to/file`
- 핵심 변경점: 한 줄 요약
- 검증 결과: 실행한 명령 + 성공/실패
- 후속 필요 작업: 있으면 번호 목록으로 제안

## 9) 프로젝트별 커스텀 섹션
아래 항목은 이 저장소 기준으로 유지한다.

### 기술 스택
- React
- Vite
- Node.js

### 자주 쓰는 명령어
- 설치: `npm install`
- 개발: `npm run dev`
- 린트: `npm run lint`
- 빌드: `npm run build`

### 도메인 규칙
- 도메인별 화면/경로/네이밍은 문서에 명시된 기준을 우선
- 도메인 특화 구조는 일반 가이드에 예시로만 기록하고, 필요 시 프로젝트 문서에서 확정

## 10) 작업별 상세 가이드 연결
- 프론트엔드: `docs/commands/frontend.md`
- 백엔드: `docs/commands/backend.md`
- 배포: `docs/commands/deploy.md`
- 코드리뷰: `docs/commands/review.md`
- 초기 세팅(일반): `docs/commands/initial-setup.md`

## 11) 기본 실행 규칙
- 사용자가 "초기 구조 만들어줘", "초기 세팅 해줘", "라우터 붙여줘"와 같은 요청을 하면, `docs/commands/initial-setup.md`를 우선 적용
- 동일 요구사항 재지시가 없어도 일반 초기 세팅 가이드를 기준으로 스캐폴딩
- 사용자가 구조를 명시하면 사용자 요구를 우선
